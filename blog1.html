Getting started Cloning and experimenting with open-source projects on Git-Hub

28th November 2020

Firstly lets clear out the air on what the difference is between git and github. Git is a version control software that lets you manage and keep track of the source code history in short it keeps a track of your development somewhat like individual lego bricks, you can always go back and change a block and add new one to the main product if you are not happy with things.GitHub on the other hand is a cloud based hosting service that employs this git mechanism and helps you manage git repositories, it makes collaborating with many people easier


Here is a nice short and detailed video explaining it


Cloning different open source progects

Firstly headover to Github and try to find beginner friendly open-source repositories that you would like to clone and experiment with, once found here are the steps that you need to follow.

  1. Fork the repo

    2. Find the repo you want to clone, open it and then click on the fort button on the top right corner

  2. Clone the repo

    3. To clone this repo click on the green code button and copy the HTTPS url

    Now using the terminal go to the folder where you want to download this file

    once there type in the command git clone (repo-url) replace repo-url with the HTTPS url link that you have already copied

  3. Create a branch

    4. Now we create a new branch to the main branch using the command git branch <'branch-name'>, we will be adding our new features or making changes on this new branch

  4. Add and Commit changes

    5. Once we have added our features or made the changes we now have to add using the command git add . and then commit the changes using the command git commit -m "The message of our changes"

  5. Run and play with the code

    6. Now as you have added new features to the program you can run it using an editor and see your feature come to life if nothing goes wrong. This is a good way to experiment and understand how various codes and apps work.


    Takeaway

    While this a good way to try out different projects and codes please do not forget to try and implement projects on your own as ultimately this is what will help you learn and gain experience with your muscle memory which in tern will help you to write code on your own.
